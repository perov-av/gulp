Старт нового проекта:
1. Клонируем сборку из https://github.com/perov-av/gulp
2. Удаляем файл .gitignore и папку .git
3. Создаем новую папку с проектом и копируем туда скачанные ресурсы.
4. Открываем через редактор и открываем консоль
5. Подаем команду в консоли "npm i" - устанавливаются все пакеты и зависимости из package.json
6. Запускаем сборку, чтобы отобразить папку public командой "gulp build"
7. Запускаем команду "git init" для инициализации git
8. Добавляем в .gitignore папку node_modules
9. Создаем файл README.md командой "git add README.md"
10. Проверяем статус папок командой "git status". Результат - они все красные.
11. Запускаем команду "git add ."
12. Проверяем статус папок командой "git status". Результат - они стали зеленые.
13. Делаем первый коммит командой " git commit -m "Пробный комит" "
14. Переименовываем ветку с master на  main командой "git branch -M main"
15. На Гитхабе создаем репозиторий. Копируем от туда команду для связи локальной папки 
		с данным репозиторием. Команда должна иметь вид "git remote add origin https://github.com/perov-av/GlinSerg.git"
		только в конце наименование созданного репозитория.
16. Подаем команду "git push -u origin main"
17. Обновляем страницу на Гитхабе и видим, что наши папки и файли залились в репозиторий Гитхаба
18. Подаем команду "git subtree push --prefix public origin gh-pages". Она создает ветку, содержащуу только
		папку public и делает эту ветку доступной для Гитхаб Пэйджс.
19. Заходим в репозиторий на Гитхаб и убеждаемся, что там теперь 2 ветки: main и gh-pages
20. Переходим во вкладку Settings
21. В левом меню переходим на Pages. В поле Sourse должно быть выставлено наименование Deploy from a branch
		А ниже в поле Branch выставляем нашу ветку gh-pages
22. Переходим по ссылке, которая указана выше на вкладке pages. Результат - на ней отображается проект с папки public.
